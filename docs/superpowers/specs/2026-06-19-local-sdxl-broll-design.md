# Local SDXL B-Roll Generation Design

**Date:** 2026-06-19
**Status:** Awaiting approval
**Author:** brainstormed with Cody
**Parent spec:** [2026-05-30-youtube-automation-pipeline-design.md](2026-05-30-youtube-automation-pipeline-design.md) — replaces the Pexels-only Media Agent with a local-diffusion-primary, Pexels-fallback design.

## 1. Goal

Replace generic Pexels stock footage with locally generated SDXL images animated via ffmpeg Ken Burns, so b-roll matches each script beat instead of being keyword-matched stock. Pexels remains as a per-scene safety net so a broken ComfyUI never kills a run.

Target hardware: RTX 5070, 12GB VRAM. Target content: abstract / informational videos (history, science, explainers) where there are no recurring characters or real-world events.

## 2. Decisions locked in during brainstorming

| Decision | Choice |
|---|---|
| Content type assumption | Abstract / informational only. No recurring characters, no real-world / news footage. |
| Visuals strategy | Stills + ffmpeg Ken Burns (zoompan). No SVD / AnimateDiff in scope. |
| Image model | SDXL base 1.0 only. No Flux, no LoRAs, no upscaling. Model swap deferred. |
| ComfyUI install location | `third_party/ComfyUI/` inside the project, gitignored. Self-contained per user preference. |
| ComfyUI process management | User starts ComfyUI manually in a separate terminal. Pipeline does not spawn or monitor the process. |
| Pexels fate | Kept as fallback. Default `media_source` is `local_diffusion`; Pexels triggers automatically on per-scene generation failure. |
| Style consistency | Single `video_style` string at script level, generated by Gemini, appended to every scene's `image_prompt`. Not user-input. |

## 3. Architecture

One new client, one new ffmpeg helper, a new source-strategy interface, a rewritten Media Agent, and an extension to the Script Agent's visuals prompt. No changes to Voice, Caption, Render, or Upload agents. No changes to pipeline contracts other than the `script.json` scene shape.

```
┌─────────────────────────┐     ┌──────────────────────────────┐
│  agents/script.py       │     │  agents/media.py             │
│  - emits image_prompt   │     │  MediaAgent (rewritten)      │
│  - emits video_style    │────▶│  per scene:                  │
│  - emits pexels_query   │     │    primary  = LocalDiffusion │
│    (fallback)           │     │    fallback = Pexels         │
└─────────────────────────┘     │  on per-scene failure → fb   │
                                └──────┬───────────────┬───────┘
                                       │               │
                          ┌────────────▼──────┐  ┌─────▼────────────┐
                          │ LocalDiffusion    │  │ PexelsSource     │
                          │ Source            │  │ (extracted from  │
                          │ - ComfyUI client  │  │  current code)   │
                          │ - Ken Burns       │  └──────────────────┘
                          └─────┬─────────────┘
                                │
                  ┌─────────────▼──────────────┐  ┌──────────────────────┐
                  │ clients/comfyui.py         │  │ ffmpeg/ken_burns.py  │
                  │ - submit workflow JSON     │  │ - still_to_clip()    │
                  │ - poll /history            │  │ - zoompan filter     │
                  │ - download PNG             │  │ - randomized motion  │
                  └────────────────────────────┘  └──────────────────────┘
```

## 4. Script Agent change

[script_meta.py](src/yt_auto/prompts/script_meta.py) is extended so the scene-visuals prompt now asks Gemini to produce, for each scene:

- `pexels_query` — kept as today (fallback path).
- `image_prompt` — descriptive SDXL-style prompt for the scene's visual content (subject, composition, lighting, mood). 1–3 sentences, no style boilerplate.

Plus one new script-level field:

- `video_style` — a single string applied to every scene, e.g. *"cinematic photography, 35mm film, dramatic natural lighting, shallow depth of field, photorealistic"*. Generated by Gemini in the same call, derived from `voice_category` and `topic` so the choice reflects the tone of the video.

At image-generation time, `LocalDiffusionSource` concatenates `f"{image_prompt}, {video_style}"` before submitting to ComfyUI. This guarantees stylistic coherence across the 10-30 scenes in a video instead of jarring style mismatches.

`script.json` schema delta (additive only):
```json
{
  "video_style": "string",
  "scenes": [
    {
      "...existing fields...": "...",
      "pexels_query": "string",
      "image_prompt": "string"
    }
  ]
}
```

## 5. New client: `clients/comfyui.py`

Thin async HTTP wrapper around ComfyUI's REST surface. Single public method:

```python
class ComfyUIClient:
    def __init__(self, base_url: str, *, timeout_s: float = 180.0) -> None: ...

    async def generate_image(
        self,
        *,
        prompt: str,
        width: int,
        height: int,
        seed: int,
        dest: Path,
    ) -> None:
        """Submit a text2img workflow, poll until done, download PNG to `dest`."""
```

Internals:
- Loads a fixed workflow template at module import time from `src/yt_auto/clients/workflows/sdxl_txt2img.json` — checked into the repo.
- Template uses placeholders for `prompt`, `width`, `height`, `seed`; no other knobs exposed.
- Sampler / steps / CFG hardcoded in the template (sensible SDXL defaults: euler, 25 steps, CFG 7).
- POSTs to `/prompt`, polls `/history/{prompt_id}` until the output node has an image, then GETs `/view?filename=...&type=output` and writes to `dest`.
- Raises `ComfyUIError` on HTTP failure, timeout, or missing output. Never retries internally — retry is the Media Agent's job.

Caller's responsibility: pass SDXL-native dimensions (close to 1024×1024 area, multiples of 64). Asking SDXL base for 1920×1080 directly produces broken compositions. Generation dims are not the video output dims — `LocalDiffusionSource` picks them, Ken Burns handles the upscale + crop to final frame.

## 6. New ffmpeg helper: `ffmpeg/ken_burns.py`

```python
async def still_to_clip(
    *,
    src: Path,
    dest: Path,
    duration_s: float,
    width: int,
    height: int,
    fps: int,
    seed: int,
) -> None
```

Output shape (codec, container, dimensions, fps) is identical to what [prepare_clip.py](src/yt_auto/ffmpeg/prepare_clip.py) produces, so the existing `concat_clips` is unchanged.

The source PNG is typically smaller than the output frame (SDXL-native ~1024² vs. 1920×1080 / 1080×1920 output). The `zoompan` filter handles both upscaling and motion in a single pass — it samples the source at sub-pixel positions and writes the requested output dims, so there is no separate upscale step. Effective minimum zoom is therefore >1 (enough to cover the output canvas at the source's narrower aspect), and motion ranges are computed from the source/output dim ratio.

Motion variety: `seed` deterministically picks one of four motion presets — `zoom_in`, `zoom_out`, `pan_left`, `pan_right`. Same seed → same motion (reproducibility for re-runs).

## 7. Media Agent rewrite

Introduces a `SceneSource` protocol:

```python
class SceneSource(Protocol):
    async def produce_clip(
        self,
        *,
        scene: dict[str, Any],
        target_duration_s: float,
        width: int,
        height: int,
        fps: int,
        dest: Path,
    ) -> None: ...
```

Two implementations:

- `LocalDiffusionSource(comfyui_client, video_style)` — picks SDXL-native gen dims for the target aspect ratio (long → 1344×768, short → 768×1344), generates PNG via ComfyUI, runs Ken Burns to `dest`. Raises `SceneSourceError` on any failure.
- `PexelsSource(pexels_client)` — wraps the existing search → download → `prepare_clip` flow extracted from today's [media.py](src/yt_auto/agents/media.py). Raises `SceneSourceError` on any failure.

Rewritten `MediaAgent.run`:

1. At startup, if `media_source == "local_diffusion"`, ping ComfyUI's `/system_stats`. If unreachable, log a single warning and downgrade the entire run to Pexels-only (no per-scene fallback attempts — saves N timeout waits).
2. For each scene, try `primary.produce_clip(...)`. On `SceneSourceError`, log it and try `fallback.produce_clip(...)`. If both fail, raise `MediaError` and abort the run (matches current behavior — Pexels failure is fatal today too).
3. Concat as today.

Result: a single scene's bad luck (ComfyUI OOM, bad seed timeout) does not waste the run; a structural problem (ComfyUI not running) degrades gracefully to Pexels.

## 8. Install / setup

New doc: `docs/comfyui-setup.md`. Contents:

- `git clone https://github.com/comfyanonymous/ComfyUI third_party/ComfyUI`
- Create a venv inside ComfyUI's directory (separate from the project venv — different torch).
- `pip install -r requirements.txt` plus the torch CUDA wheel matching the 5070.
- Download `sd_xl_base_1.0.safetensors` from HuggingFace into `third_party/ComfyUI/models/checkpoints/`.
- Launch line: `python main.py --listen 127.0.0.1 --port 8188`.
- Smoke test: `curl http://127.0.0.1:8188/system_stats`.

`.gitignore` gets `third_party/`.

Config additions in [config.py](src/yt_auto/config.py):

- `comfyui_url: str = "http://127.0.0.1:8188"`
- `media_source: Literal["local_diffusion", "pexels"] = "local_diffusion"`

No new required env vars. ComfyUI URL is overridable via env for tests / CI.

## 9. Error handling

| Failure | Behavior |
|---|---|
| ComfyUI unreachable at run start | One warning log, force whole run to Pexels. No per-scene fallback attempts. |
| ComfyUI per-scene timeout / 500 / OOM | Log error with scene index, fall back to Pexels for that scene only, continue. |
| Pexels per-scene failure (after fallback) | `MediaError`, run aborts. Same as today. |
| Pexels per-scene failure (when Pexels is primary because of run-start downgrade) | `MediaError`, run aborts. Same as today. |
| Ken Burns ffmpeg failure | Treated as a scene-source failure, falls back to Pexels per above. |

No retries inside any client. The fallback is the retry strategy.

## 10. Testing

Unit (deterministic, no network):

- `ComfyUIClient` workflow serialization: given prompt/dims/seed, the JSON sent to `/prompt` matches the expected shape. HTTP mocked.
- `ComfyUIClient` polling: simulated `/history` responses (pending → done), verify download is called with the right filename.
- `LocalDiffusionSource.produce_clip` with a fake ComfyUI client returning a fixture PNG → asserts the resulting clip duration matches target (via `probe_duration_s`) and dimensions are correct.
- `MediaAgent` fallback: primary raises `SceneSourceError` on scene 2 only, fallback returns OK → assert both sources were called, video has correct number of clips.
- `MediaAgent` startup downgrade: ComfyUI ping fails → assert primary is never invoked, all scenes go through Pexels.

Manual:

- One full short-format run end-to-end. Eyeball video. Compare to a Pexels-only baseline of the same script.

## 11. Out of scope

Deliberately deferred — all addable later without redesign because the `SceneSource` interface and config switch isolate the change:

- SVD / AnimateDiff (real animated clips).
- Flux model support.
- LoRA loading or style training.
- Upscaling (Real-ESRGAN etc).
- Face / character consistency across scenes.
- Automatic ComfyUI process management (subprocess, healthcheck loop, auto-restart).
- Per-run media-source override on the CLI.
- Model swapping mid-pipeline.
- GPU memory monitoring.
